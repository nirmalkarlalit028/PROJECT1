    • Variable's

    INFORMATION:- A JavaScript variable is a named container for storing data values that can be used and potentially modified throughout a program. Variables are declared using the keywords let, const, or var .

    1. var (The Old Way)
    Introduced at the beginning of JavaScript, var is now generally avoided in modern development due to its unpredictable behavior.
    1. Function: Scoped: If declared inside a function, it's available anywhere in that function. If declared outside, it's global.
    2. Hoisting: You can use a var variable before its declaration line without an error; it will just return undefined.
    3. Redeclaration: You can redeclare the same variable name multiple times in the same scope without the engine complaining.
    2. let (The Modern Standard)
    Introduced in ES6 (2015), let is the preferred choice for variables that need to change.
    1. Block Scoped: It only exists within the nearest pair of curly braces {} (like in an if statement or a for loop).
    2. Temporal Dead Zone: While technically hoisted, you cannot access it before the line it is declared on, or you will get a ReferenceError.
    3. No Redeclaration: You cannot declare the same variable twice in the same scope using let.
    3. const (For Constants)
    Also introduced in ES6, const is used for values that should not be reassigned.
    1. Immutable Binding: Once you assign a value, you cannot point the variable to a new value (e.g., const x = 1; x = 2; throws an error).
    2. Object Mutation: While you can't reassign the variable itself, if it holds an array or object, you can still modify the properties inside that object.
    3. Required Initializer: You must assign a value at the exact moment you declare it.
